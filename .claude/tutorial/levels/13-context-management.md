## 目标

学会管理对话上下文，理解 `/compact`、`/clear`、`/btw` 的区别和使用时机，并知道什么时候该用子 Agent 来减轻主上下文负担。

## 任务

随着对话变长，消耗的 token 越来越多，响应速度和费用都会受到影响。Claude Code 提供了多种工具来管理上下文。

1. 确保你已经有了一段较长的对话（至少和 Claude 来回聊了 5 轮以上）
2. 输入 `/compact` 压缩对话上下文
3. 观察压缩后发生了什么变化
4. 了解什么时候应该用 `/compact`，什么时候应该用 `/clear`
5. 了解 `/btw` 适合什么场景：临时问一个旁路问题，但不污染主对话上下文
6. 思考什么时候应该把调研交给子 Agent，而不是让主对话读取大量文件

完成后告诉我：`/compact`、`/clear`、`/btw` 分别适合什么场景？子 Agent 如何帮助管理上下文？

## 验证

告诉我：

- `/compact` 和 `/clear` 的区别和使用场景
- `/btw` 适合什么场景
- 子 Agent 如何帮助减少主对话上下文压力

## 提示

- 长对话会消耗大量 token，影响响应速度
- `/compact` 压缩对话但保留关键信息，对用户基本透明
- `/clear` 完全清除对话历史，适合切换到不相关的新任务
- `/btw` 适合临时旁路问题，不把问答写入主上下文
- 大范围调研、搜索、代码审查可以交给子 Agent，避免主对话塞满文件内容
- Claude Code 在接近上下文限制时会自动触发 AutoCompact
- 如果对话太长了，试试 `/compact`；如果任务已经换了，考虑 `/clear`
