## 目标

理解 Claude Code 的六层安全防线——如何保护用户免受误操作和恶意攻击。

## 背景

作为能在终端执行命令的 AI 助手，安全至关重要。Claude Code 设计了纵深防御体系，BashTool 本身就有 9,700 行安全代码。

## 代码节选

```
Claude Code 的六层安全防线：

第一层: 配置规则
  用户预设的 allow/deny 规则
  在 settings.json 中配置
  例如：允许 npm 命令，禁止 rm -rf

第二层: 工具自检
  每个工具自我检查参数合法性
  拒绝明显不合理的输入

第三层: 小模型分类器
  快速分类请求的风险等级
  低风险自动放行，高风险进入下一层

第四层: 协调者门控
  Coordinator 审批高风险操作
  适用于多代理场景

第五层: Bash 安全分类器
  22 个独立验证器，逐项检查命令
  包括：命令注入检测、路径穿越检测、
  敏感文件保护、危险命令黑名单等

第六层: 用户确认
  最终人工确认
  高风险操作必须用户点头
```

**BashTool 22 个验证器示例**：
- 命令注入检测
- 路径穿越检测
- 敏感文件保护（.env, credentials, .ssh）
- 危险命令黑名单（rm -rf /, dd if=）
- 网络连接限制
- 超时控制
- 参数合法性检查

## 问题

1. 六层安全防线分别是哪六层？从外到内排列。
2. 为什么需要这么多层安全？只靠"用户确认"一层不行吗？
3. BashTool 的 22 个验证器主要防护哪些类型的风险？

## 期望答案

1. 配置规则 → 工具自检 → 小模型分类器 → 协调者门控 → Bash 安全分类器 → 用户确认
2. 不行——用户可能不注意确认、需要防御提示注入攻击、需要分层防御纵深（defense in depth）、自动化场景需要自动判断
3. 命令注入、路径穿越、敏感文件泄露、危险命令执行、资源滥用等
