## 目标

理解 Claude Code 中子 Agent 的作用，学会把一个大任务拆成单独的子任务，交给专门的 agent 去处理。

## 概念

### 什么是子 Agent？

子 Agent 是 Claude 临时拉起的专门帮手，用来处理一个明确的子任务，比如调研、搜索、代码修改、测试验证等。

### 它适合做什么？

- **并行调研** — 同时探索多个方向
- **局部搜索** — 只关注一个模块或一类文件
- **任务分包** — 主对话负责协调，子 Agent 负责执行

### 它和主对话的区别

主对话负责综合与决策；子 Agent 负责完成一个更聚焦的子任务，然后把结果带回来。

### 它和 Agent Teams 的区别

- **子 Agent**：一次性任务分包
- **Agent Teams**：多个队友持续协作、彼此通信

## 任务

1. 输入 `/help`，看看你当前运行时里是否有 `/agent` 或相关 agent 命令
2. 让 Claude 帮你把一个任务拆给子 Agent，例如：
   - "请用一个 agent 帮我调研这个项目的主要模块，再汇总给我"
3. 观察 Claude 是如何把任务分出去，又如何把结果带回主对话的
4. 总结一下：什么任务适合交给子 Agent？

完成后告诉我：
- 子 Agent 更适合处理什么类型的任务
- 它和主对话的分工有什么不同

## 验证

告诉我：
- 子 Agent 的作用是什么
- 举 1-2 个适合交给子 Agent 的任务
- 它和 Agent Teams 的区别是什么

## 提示

- 子 Agent 更适合目标清晰、可以独立完成的子任务
- 它是"任务分包"，不是"团队长期协作"
- 命令名称以你 `/help` 实际看到的内容为准
